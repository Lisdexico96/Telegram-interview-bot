================================================================================
TELEGRAM INTERVIEW BOT - RAILWAY DEPLOYMENT STEPS
================================================================================

Follow these steps to deploy your bot on Railway for 24/7 operation:

================================================================================
STEP 1: PREPARE YOUR CODE
================================================================================

1. Make sure all your code is committed to GitHub:
   - Open terminal/command prompt
   - Navigate to your bot directory
   - Run: git add .
   - Run: git commit -m "Ready for Railway deployment"
   - Run: git push origin main

   (If you haven't set up git yet, see STEP 1A below)

STEP 1A: If you haven't pushed to GitHub yet:
   - Go to github.com and create a new repository
   - Copy the repository URL
   - In your bot directory, run:
     git init
     git add .
     git commit -m "Initial commit"
     git branch -M main
     git remote add origin YOUR_REPO_URL
     git push -u origin main


================================================================================
STEP 2: CREATE RAILWAY ACCOUNT
================================================================================

1. Go to https://railway.app
2. Click "Start a New Project" or "Login"
3. Sign up with GitHub (recommended) or email
4. Complete the signup process


================================================================================
STEP 3: CREATE NEW PROJECT ON RAILWAY
================================================================================

1. After logging in, click "New Project"
2. Select "Deploy from GitHub repo"
3. If prompted, authorize Railway to access your GitHub account
4. Find and select your repository from the list
5. Click "Deploy Now"


================================================================================
STEP 4: CONFIGURE ENVIRONMENT VARIABLES
================================================================================

1. In Railway dashboard, click on your newly created service
2. Click on the "Variables" tab (or "Settings" â†’ "Variables")
3. Click "New Variable" or "Add Variable"
4. Add the first variable:
   - Name: BOT_TOKEN
   - Value: YOUR_TELEGRAM_BOT_TOKEN
   - Click "Add" or "Save"

5. Add the second variable:
   - Name: ADMIN_CHAT_ID
   - Value: YOUR_TELEGRAM_CHAT_ID
   - Click "Add" or "Save"

IMPORTANT NOTES:
- Do NOT include quotes around the values
- For BOT_TOKEN: Use the token from @BotFather (format: 123456789:ABCdef...)
- For ADMIN_CHAT_ID: Use your chat ID from @userinfobot (just the number)
- If you have multiple admin IDs, separate with commas: 123456789,987654321


================================================================================
STEP 5: WAIT FOR DEPLOYMENT
================================================================================

1. Railway will automatically start building and deploying your bot
2. Go to the "Deployments" tab to see progress
3. Wait for status to show "Active" or "Success" (usually 2-3 minutes)
4. You'll see build logs showing:
   - Installing dependencies
   - Building the application
   - Starting the bot


================================================================================
STEP 6: VERIFY YOUR BOT IS RUNNING
================================================================================

1. Go to the "Logs" tab in Railway
2. You should see messages like:
   "Initializing bot..."
   "Bot is starting..."
   "Bot token: 8108233182..."

3. Test your bot on Telegram:
   - Open Telegram
   - Find your bot (search for the username you gave it)
   - Send /start
   - The bot should respond asking for your name

4. If the bot responds, SUCCESS! Your bot is now running 24/7


================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: Bot not responding on Telegram

SOLUTION 1: Check Railway Logs
   - Go to Railway dashboard â†’ Logs tab
   - Look for error messages
   - Common errors:
     * "BOT_TOKEN not found" â†’ Environment variable not set correctly
     * "Network error" â†’ Temporary issue, will retry automatically
     * "Conflict error" â†’ Another instance running (unlikely on Railway)

SOLUTION 2: Verify Environment Variables
   - Go to Variables tab
   - Make sure BOT_TOKEN and ADMIN_CHAT_ID are both set
   - Check for typos or extra spaces
   - Remove and re-add if needed

SOLUTION 3: Redeploy
   - Go to Deployments tab
   - Click "Redeploy" on the latest deployment
   - Wait for it to complete

PROBLEM: Build failed

SOLUTION:
   - Check the build logs in Railway
   - Make sure requirements.txt is correct
   - Verify all files are pushed to GitHub
   - Try redeploying

================================================================================
UPDATING YOUR BOT
================================================================================

1. Make changes to your code locally
2. Commit and push to GitHub:
   git add .
   git commit -m "Your update description"
   git push origin main

3. Railway will automatically detect the change and redeploy
4. Wait 2-3 minutes for deployment to complete
5. Check Logs to verify the update is working


================================================================================
STOPPING YOUR BOT
================================================================================

1. Go to Railway dashboard
2. Click on your service
3. Go to Settings tab
4. Click "Delete Service" (or pause if available)

Note: Your bot will stop running and you'll be charged only for the time it was active.
